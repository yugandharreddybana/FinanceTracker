Current problem: Node server stores data in-memory (lost on restart) and

In server/routes/finance.ts — replace the in-memory storage with

```ts
import fetch from 'node-fetch'; // or use built-in fetch (Node 18+)

const BACKEND_URL = process.env.VITE_API_URL || 'http://localhost:8080';

router.get('/transactions', async (req, res) => {
  try {
    const response = await fetch(`${BACKEND_URL}/api/finance/transactions`, {
      headers: { 
        Authorization: req.headers.authorization || '',
        'Content-Type': 'application/json'
      }
    });
    const data = await response.json();
    res.status(response.status).json(data);
  } catch (err) {
    res.status(502).json({ error: 'Backend unavailable' });
  }
});
// Repeat this pattern for ALL routes (DELETE, PUT, POST)
// DELETE must forward the id correctly: DELETE /transactions/:id → backend DELETE /api/finance/transactions/:id
```

### FIX 4: FIX CORS IN SERVER
Replace:
```ts

```
With:
```ts
const allowedOrigins = (process.env.ALLOWED_ORIGINS || 'http://localhost:3000')
  .split(',')
  .map(o => o.trim());

app.use(cors({
  origin: (origin, callback) => {
    if (!origin || allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error('Not allowed by CORS'));
    }
  },
  credentials: true,
}));
```
